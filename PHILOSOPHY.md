# Model philosophy (first principles)

1. **Start from a decision, not a model.** What decision, who acts on it,
   what does each error cost? The error-cost asymmetry IS the spec.
   (Ours: missed finish page = wrong bid; extra kept page = one glance.)
2. **Define "done" as a number before building.** Our contract: zero missed
   finish pages, most junk cut, on never-seen projects. No number, no ship.
3. **Intelligence is a cost. Buy the dumbest thing that clears the bar.**
   Climb the rung ladder only when a MEASURED constraint forces it.
   Best model is no model: derive with rules wherever rules work (keep ←
   category).
4. **Spend on data; models are disposable.** Labels compound; models are
   recomputed in minutes. The labeling factory outvalues any model it makes.
5. **Test like reality; fix one named constraint per cycle.** Eval on held-
   out permits (tomorrow's upload). Each cycle: measure → prove the binding
   constraint → cheapest attack (see .claude/skills/diagnose-model) → repeat.

New feature = new decision (1) + its number (2) + cheapest specialist (3)
+ labels from the same factory (4) + reality-shaped test (5).
